greet.esp un cli en 6 líneas 
# greet.esp
cmd greet do
  arg name: str
  flag loud: bool = false

  "Hola, #{name}!" |> when(loud, upcase) |> print
end

$ espeto run greet.esp -- Mundo --loud

~/espeto ❯

Menos ambigüedad, más precisión.

Un lenguaje de programación pequeño, pipe-céntrico y estructurado para que los modelos de IA lo escriban con precisión.

Herramientas de consola en minutos.

En Espeto, program, cmd, arg y flag son palabras clave del lenguaje. La validación, la página --help, --version y el despacho de subcomandos se generan solos.

  • Un CLI completo en menos de 15 líneas.
Todo.esp varios subcomandos 
program todo do
  desc "todo manager"
  version "0.1.0"

  cmd add do
    arg item: str
    "added: #{item}" |> print
  end

  cmd remove do
    arg id: int
    "removed: #{id}" |> print
  end
end

$ espeto run todo.esp -- --help
Usage: todo <command> [options]
Commands: add, remove

Diseñado para IAs (y humanos).

Espeto no tiene magia oscura ni azúcar sintáctico innecesario. Existe una sola forma canónica de escribir las cosas. Esto reduce drásticamente las alucinaciones cuando le pides a ChatGPT o Claude que escriban scripts por ti.

Sin coerciones implícitas 
# Fallo duro: no se mezclan tipos
1 + 1.0  # ERROR

# Igualdad estructural estricta
1 == 1.0 # false

# Truthiness explícito
if "texto" do # ERROR (requiere bool)
if length(x) > 0 do ... # CORRECTO

Todo fluye hacia adelante.

El operador |> no es azúcar sintáctico, es la base del lenguaje. Cualquier dato pasa como primer argumento a la función de su derecha. No necesitas anidar llamadas ni declarar variables temporales.

  • Lectura de arriba a abajo.
  • Soporta lambdas anónimas en línea.
Ejemplo.esp 
# Desde una simple cadena...
"  sardinas frescas  "
  |> trim
  |> upcase
  |> print
# SARDINAS FRESCAS

# Hasta transformaciones de listas
users
  |> filter(fn u => u.active)
  |> sort_by(.age)
  |> map(.name)
  |> print

Todo lo necesario, ya incluido.

Más de 80 funciones listas para usar sin necesidad de imports. JSON, shell (sh/sh!), streams lazy de líneas, archivos, strings y matemáticas — todo accesible globalmente y optimizado para usarse con el pipe.

users.json -> cli 
# try captura errores devolviendo 
# o ejecutando el bloque rescue
data = try "users.json" 
  |> read 
  |> parse_json
rescue err -> 
  raise("Error crítico: #{err}")

data |> map(.email) |> print

Un binario, cero dependencias.

espeto build empaqueta tu programa .esp + el intérprete en un único ejecutable autocontenido. La máquina destino no necesita Node, ni Bun, ni Espeto — copias el binario y se ejecuta. Por debajo usa Bun con su flag --compile, así que el cross-compile a las plataformas principales es un único flag.

  • Un solo archivo: imports relativos resueltos al empaquetar.
  • Cross-compile a darwin/linux/windows · arm64/x64.
terminal ship it 
# empaqueta tu cli en un binario standalone
$ espeto build hola.esp -o hola
$ ./hola Mundo --loud
HOLA, MUNDO!

# cross-compile y deploy
$ espeto build hola.esp -o hola --target linux-arm64
$ scp hola server:/usr/local/bin/

# targets: darwin-arm64 · darwin-x64
#          linux-arm64 · linux-x64 · windows-x64

Spec · v0

El lenguaje, en una página.

Quince decisiones cerradas. Una sola forma canónica para cada cosa.

Top-level
import, def/defp, cmd, program. Un cmd o program por fichero.
cmd block
do/end. Orden: meta (desc/version) → declaraciones (arg/flag) → body.
program block
Agrupa varios cmd como subcomandos con flags compartidas. --help/--version auto.
Bindings
x = expr (sin let). Rebinding sí. Valores inmutables.
Pipe |>
First-arg piping. RHS: llamada / nombre pelado / lambda / .field. Placeholder _ para reposicionar el LHS.
Funciones
def f(x) = expr (one-liner) o def f(x) do … end (bloque). defp para privadas.
Lambdas
fn x => expr, fn(x,y) => expr, fn() => expr. Solo expresión.
Errores
Excepciones con raise. Auto-rescue en cmd. try expr rescue err -> … para local.
Módulos
Fichero = módulo. Imports pelados: import "./x" only [a, b as c].
Source bindings
__file__ / __dir__ auto-inyectados por módulo (definition-site, closure-captured).
Tipos
int, float, str, bool, nil, list, map, fn. Sin coerción.
Igualdad
== único, estructural. 1 == 1.0 es false.
Truthiness
Estricto: solo bool en if. and/or/not requieren bool.
Control
Solo if/else if/else do…end, expresión-valuada. Sin ternario, sin case en v0.
Strings
"…" con #{x}. Comentarios #; doc-comments ## sobre def/defp. (Multilínea """…""" planeado para v1.)
Concat
Solo funciones (concat, join). Sin operadores <>/++.
Stdlib
~80 funciones auto-cargadas. snake_case. ? en predicados. Sin loops. Síncrono.

Errores

+ variantes
raise try_*

Stdlib raisea por defecto. Las try_* devuelven {ok, value} estilo Result.

Aún no en v0

pattern matching · regex · HTTP · fechas · async · tuples · atoms · TCO garantizado. Para v1+.

Roadmap

10 hitos completados.

Cada hito fue "el primer programa que corría end-to-end". v0.1.0 cierra los diez. Después siguen build, watch, LSP, multi-subcomando y más.

02

Encadenamiento

"x" |> upcase |> print — más builtins, chains de pipes.

03

El primer CLI completo

hola.esp — cmd, arg, flag, interpolación, when.

06

Post-v0.1.0 · en main

Lo que vino después.

Features post-spec que ya están en main, mientras camino hacia v0.2 y npm publish.

espeto build

Empaquetado standalone vía Bun --compile. Cross-target a darwin/linux/windows.

espeto run --watch

Re-ejecución on-change. Sigue corriendo entre errores.

espeto lsp + VS Code

Servidor LSP por stdio + extensión: hover, go-to-def, diagnostics, completion, refs/rename, outline, signature help, semantic tokens.

program block

Multi-subcomando con flags compartidas, --help/--version auto.

Pipe placeholder _

Reposicionar el LHS en cualquier argumento: 6 |> div(30, _).

__file__ / __dir__

Paths source-relative por módulo (definition-site, closure-captured).

sh / sh!

Comando shell vía /bin/sh -c. Result-shape o bang que devuelve stdout y lanza al fallar.

Streaming pipes

read_lines, sh_lines, stdin_lines — I/O lazy por líneas; map/filter/take aceptan streams.

espeto test

Runner para ficheros *_test.esp. Bloques test "…" do … end con assert introspectivo.

espeto docs

Imprime la referencia completa del lenguaje en markdown a stdout — pensada para pegársela a un LLM como spec.

Doc-comments ##

Líneas ## … sobre un def/defp se atachan como markdown y se renderizan en el hover del LSP.

¿Por qué "Espeto"?

Una brocheta de sardinas.

"Espeto", en el sur de España, es la brocheta donde se ensartan las sardinas para asarlas a la brasa.

Y un programa en Espeto tiene esa forma: una línea de elementos enhebrados por |>. La metáfora se queda en la silueta del código — no en lo que pasa con los datos.

Un nombre con sabor a Mediterráneo le sienta bien a un lenguaje pensado con cariño.
