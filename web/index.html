~/espeto ❯

Menos ambigüedad, más precisión.

Un lenguaje de programación pequeño, pipe-céntrico y estructurado para que los modelos de IA lo escriban de forma perfecta, sin alucinaciones.

Todo fluye hacia adelante.

El operador |> no es azúcar sintáctico, es la base del lenguaje. Cualquier dato pasa como primer argumento a la función de su derecha. No necesitas anidar llamadas ni declarar variables temporales.

  • Lectura de arriba a abajo.
  • Soporta lambdas anónimas en línea.
Ejemplo.esp 
# Desde una simple cadena...
"  sardinas frescas  " 
  |> trim 
  |> uppercase 
  |> print
# SARDINAS FRESCAS

# Hasta transformaciones de listas
users
  |> filter(fn u => u.active)
  |> sort_by(.age)
  |> map(.name)
  |> print

Herramientas de consola en minutos.

El 90% de los lenguajes te obligan a importar librerías pesadas para parsear argumentos. En Espeto, cmd, arg y flag son palabras clave del lenguaje. La validación y la página de ayuda --help se generan solas.

Hola.esp 
cmd hola do
  arg name: str
  flag loud: bool = false

  greeting = "Hola, #{name}!"
  greeting |> when(loud, uppercase) |> print
end

--------------------------------
$ espeto run hola.esp -- --help
Usage: hola <name> [--loud]

Diseñado para IAs (y humanos).

Espeto no tiene magia oscura ni azúcar sintáctico innecesario. Existe una sola forma canónica de escribir las cosas. Esto reduce drásticamente las alucinaciones cuando le pides a ChatGPT o Claude que escriban scripts por ti.

Sin coerciones implícitas 
# Fallo duro: no se mezclan tipos
1 + 1.0  # ERROR

# Igualdad estructural estricta
1 == 1.0 # false

# Truthiness explícito
if "texto" do # ERROR (requiere bool)
if length(x) > 0 do ... # CORRECTO

Todo lo necesario, ya incluido.

Aproximadamente 60 funciones listas para usar sin necesidad de imports. Manejo de JSON, lectura de archivos, transformaciones de texto y matemáticas, accesibles globalmente y optimizadas para usarse con el pipe.

users.json -> cli 
# try captura errores devolviendo 
# o ejecutando el bloque rescue
data = try "users.json" 
  |> read 
  |> parse_json
rescue err -> 
  raise("Error crítico: #{err}")

data |> map(.email) |> print
Spec · v0

El lenguaje, en una página.

Quince decisiones cerradas. Una sola forma canónica para cada cosa.

Top-level
Solo import, def, cmd. Un cmd por fichero.
cmd block
do/end. Orden: meta (desc/version) → declaraciones (arg/flag) → body.
Bindings
x = expr (sin let). Rebinding sí. Valores inmutables.
Pipe |>
First-arg piping. RHS: llamada / nombre pelado / lambda / .field.
Funciones
def f(x) = expr (one-liner) o def f(x) do … end (bloque). defp para privadas.
Lambdas
fn x => expr, fn(x,y) => expr, fn() => expr. Solo expresión.
Errores
Excepciones con raise. Auto-rescue en cmd. try expr rescue err -> … para local.
Módulos
Fichero = módulo. Imports pelados: import "./x" only [a, b as c].
Tipos
int, float, str, bool, nil, list, map, fn. Sin coerción.
Igualdad
== único, estructural. 1 == 1.0 es false.
Truthiness
Estricto: solo bool en if. and/or/not requieren bool.
Control
Solo if/else if/else do…end, expresión-valuada. Sin ternario, sin case en v0.
Strings
"…" con #{x}. Multilínea """…""". Comentarios #.
Concat
Solo funciones (concat, join). Sin operadores <>/++.
Stdlib
~60 funciones auto-cargadas. snake_case. ? en predicados. Sin loops. Síncrono.

Errores

+ variantes

Stdlib raisea por defecto. Las try_* devuelven {ok, value} estilo Result.

¿Por qué "Espeto"?

Una brocheta de sardinas.

"Espeto", en el sur de España, es la brocheta donde se ensartan las sardinas para asarlas a la brasa.

Y un programa en Espeto tiene esa forma: una línea de elementos enhebrados por |>. La metáfora se queda en la silueta del código — no en lo que pasa con los datos.

Un nombre con sabor a Mediterráneo le sienta bien a un lenguaje pensado con cariño.
